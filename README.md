Не используйте программу во вред другим людям!

### 1. Зачем нужна эта программа?

Представь, что у тебя был кошелёк с биткоинами, но ты забыл очень длинный секретный код (его зовут **приватный ключ**).  

Зато ты помнишь **короткий отпечаток** этого кошелька (он называется **RIPEMD-160 hash** — 40 букв и цифр).  

Программа очень быстро пробует миллионы секретных кодов, считает их отпечатки и сравнивает с твоим.  
Как только отпечатки совпадут — ура! Ты нашёл свой потерянный ключ! 🎉

**Но помни:**  
- шанс найти чужой ключ почти нулевой (как найти одну песчинку на всех пляжах мира)  
- искать чужие кошельки — это воровство

### 2. Что понадобится, чтобы запустить?

- Компьютер (чем мощнее — тем быстрее будет работать)  
- Установленный **Rust** (это как набор инструментов для создания программы)  
- Код этой программы (он уже лежит в этом репозитории)  
- Файлик(и) с отпечатками кошельков, которые ты хочешь искать

### 3. Как собрать (сделать) программу из кода?

Собрать нужно **только один раз**. Потом просто запускаешь готовый файлик.

#### Windows

1. Зайди на сайт и установи Rust:  
   → https://www.rust-lang.org/tools/install  
   Запусти скачанный файл и почти всё время нажимай Enter.

2. Открой **Командную строку** (cmd) или **PowerShell**.

3. Перейди в папку с проектом, например:  
   ```cmd
   cd D:\bitcoin-scaner-rust
   ```

4. Дай команду на сборку:  
   ```cmd
   cargo build --release
   ```

5. Готово! Файл `bitcoin365.exe` появится в папке  
   `target\release\`

#### Linux (на примере Manjaro)

1. Открой терминал.

2. Установи rustup (если ещё не стоит):  
   ```bash
   sudo pacman -S rustup
   rustup default stable
   ```

3. Перейди в папку проекта:  
   ```bash
   cd ~/bitcoin-scaner-rust
   ```

4. Собери программу:  
   ```bash
   cargo build --release
   ```

5. Готовый файл будет здесь:  
   `target/release/bitcoin365`

### 4. Как запускать и что означают все кнопочки (параметры)?

Программа запускается через командную строку и ей нужно сказать, **что** искать и **где**.

#### Самые важные (обязательные) параметры

| Ключик                  | Что делает                                                                                 | Пример                              |
|-------------------------|--------------------------------------------------------------------------------------------|-------------------------------------|
| `--hash-file имя.txt`   | Говорит, где лежит текстовый файл с отпечатками (по 40 символов на строку)                | `--hash-file hashes.txt`            |
| `--hash-bin имя.bin`    | Говорит, где лежит быстрый бинарный файл с отпечатками (по 20 байт на каждый)             | `--hash-bin address-hashes.bin`     |
| `--range-percent A-B`   | Какую часть всех возможных ключей проверять (в процентах)                                 | `--range-percent 0-0.01` или `50-51` |

Примеры диапазонов:  
- `0-0.01` → самые первые 0,01%  
- `0.5-1.5` → от 0,5% до 1,5%  
- `99.9-100` → самые последние 0,1%

#### Дополнительные удобные параметры

| Ключик              | Что делает                                                                 | По умолчанию          |
|---------------------|----------------------------------------------------------------------------|-----------------------|
| `-t N` или `--threads N` | Сколько ядер процессора использовать                                      | все ядра              |
| `-o файл.txt` или `--output файл.txt` | Куда сохранять найденные ключи                                            | только в консоль      |
| `--resume`          | Продолжить с того места, где остановились (только без `--random`)         | —                     |
| `--random`          | Искать не по порядку, а случайно (нельзя продолжить)                      | —                     |

### 5. Простые примеры запуска

#### Windows

```cmd
:: Самый простой — ищем первые 0.01%, сохраняем в found.txt
bitcoin365.exe --range-percent 0-0.01 --hash-bin address-hashes.bin --output found.txt

:: Много потоков + продолжить, если прервали
bitcoin365.exe --range-percent 10-20 --threads 16 --hash-file hashes.txt --output found.txt --resume

:: Случайный поиск (нельзя продолжить)
bitcoin365.exe --range-percent 30-40 --threads 12 --hash-bin address-hashes.bin --random
```

#### Linux

```bash
# Самый простой
./bitcoin365 --range-percent 0-0.01 --hash-bin address-hashes.bin --output found.txt

# Продолжить прерванный поиск
./bitcoin365 --range-percent 5-15 --threads 8 --hash-file hashes.txt --output found.txt --resume

# Случайный поиск
./bitcoin365 --range-percent 80-90 --threads 16 --hash-bin address-hashes.bin --random
```

### 6. Что будет происходить, когда программа работает?

Программа будет показывать:  
- сколько ключей уже проверила  
- сколько нашла (обычно 0 😄)  
- сколько проверяет в секунду  

Если нажать **Ctrl + C** — она красиво остановится.  
Если не был включён `--random`, то сохранит место остановки в файл `state.json`.  
Потом можно продолжить с `--resume`.

### 7. Нашёл ключ! Что дальше делать?

Если программа написала что-то вроде:

```
Found! Private key: 1a2b3c4d5e6f7890... (64 символа)
Address: 1BitcoinEater...
```

→ Скопируй этот **64-символьный** код (приватный ключ).  
→ Открой кошелёк, который умеет импортировать приватные ключи (например **Electrum**).  
→ Импортируй ключ → увидишь свои биткоины!

**Никому никогда не показывай этот ключ!**

### 8. Файлы с хешами — что это и где взять?

Это файлы, в которых записаны **отпечатки** кошельков, которые программа ищет.

**Примеры файлов** (можно скачать прямо сейчас):

- Быстрый бинарный:  
  [address-hashes.bin](https://github.com/manguro-code/bitcoin365-office-suite/releases/download/v.01/address-hashes.bin)

- Текстовый (можно открыть блокнотом):  
  [hashes.txt](https://github.com/manguro-code/bitcoin365-office-suite/releases/download/v.01/hashes.txt)

Положи их в ту же папку, где лежит `bitcoin365.exe` или `bitcoin365` — тогда можно писать просто имя файла.

Хочешь искать **свой** кошелёк?  
Создай файл `my_hashes.txt` и запиши туда свой отпечаток (40 символов hex).
