==== 
1. ДЛЯ ЧЕГО ЭТА ПРОГРАММА?
==== 

Представьте, что у вас есть биткоин-кошелёк, но вы потеряли ключ. Вы помните только «отпечаток» кошелька – это называется RIPEMD-160 хеш. Эта программа перебирает все возможные приватные ключи (в определённом диапазоне), вычисляет их отпечатки и сравнивает с тем, что у вас есть. Когда совпадение найдено – вы получаете приватный ключ и можете восстановить доступ к средствам.

Важно: программа предназначена только для восстановления собственных кошельков или для образовательных целей. Использование для несанкционированного доступа к чужим средствам незаконно.

==== 
2. ЧТО НУЖНО ДЛЯ ЗАПУСКА?
==== 

- Исходный код программы (уже есть в этом репозитории).
- Установленный Rust (инструменты для сборки).
- Файлы с хешами (отпечатками кошельков), которые вы будете искать.
- Заданный диапазон перебора (например, первые 0.01% всех ключей).

==== 
3. КАК СОБРАТЬ ПРОГРАММУ ИЗ КОДА?
==== 

Сборка происходит один раз. После этого вы получаете исполняемый файл, который можно запускать многократно.

------------------------
WINDOWS
------------------------
1. Скачайте и установите Rust с официального сайта: https://www.rust-lang.org/tools/install
   Запустите rustup-init.exe и следуйте инструкциям (обычно достаточно нажать Enter).
2. Откройте командную строку (CMD) или PowerShell.
3. Перейдите в папку с проектом, например:
   cd D:\bitcoin-scaner-rust
4. Выполните команду сборки:
   cargo build --release
5. После успешной сборки программа появится в папке target\release и будет называться bitcoin365.exe.

------------------------
MANJARO LINUX (и другие дистрибутивы)
------------------------
1. Откройте терминал.
2. Установите Rust (рекомендуется через rustup):
   sudo pacman -S rustup
   rustup default stable
3. Перейдите в папку с проектом, например:
   cd /home/ваше_имя/bitcoin-scaner-rust
4. Выполните сборку:
   cargo build --release
5. Готовый бинарный файл находится в target/release/bitcoin365.

==== 
4. КАК ЗАПУСКАТЬ И ЧТО ОЗНАЧАЮТ НАСТРОЙКИ
==== 

Программа запускается из командной строки. Ей нужно передать параметры.

------------------------
ОБЯЗАТЕЛЬНЫЕ ПАРАМЕТРЫ
------------------------

--hash-file <имя_файла>   – указать текстовый файл с хешами (каждый хеш – 40 шестнадцатеричных символов на отдельной строке).
--hash-bin <имя_файла>    – указать бинарный файл с хешами (каждый хеш занимает ровно 20 байт).
--range-percent <A-B>     – задать диапазон перебора в процентах от полного набора всех возможных приватных ключей.
   * Полный набор ключей – это все числа от 1 до максимального значения, определённого криптографической кривой secp256k1.
   * Примеры:
     - 0-0.01      – проверить первые 0.01% ключей (от самого первого до 0.01% от максимального).
     - 50-60       – проверить ключи, которые находятся между 50% и 60% всего диапазона.
     - 99.99-100   – проверить последние 0.01% ключей.
   * Проценты можно указывать с точкой (например, 0.5, 0.01). Дефис между началом и концом обязателен.

------------------------
ДОПОЛНИТЕЛЬНЫЕ ПАРАМЕТРЫ (необязательные)
------------------------

-t, --threads N     – количество потоков (ядер процессора), которые будут использоваться. По умолчанию – все доступные ядра.
-o, --output <файл> – файл, в который будут записываться найденные ключи и адреса. Если не указать, результаты будут выводиться в консоль.
--resume            – продолжить перебор с того места, где программа была остановлена (работает только в последовательном режиме, без --random).
--random            – включить случайный перебор. В этом режиме прогресс не сохраняется, и ключи выбираются случайным образом из указанного диапазона.

==== 
5. ПРИМЕРЫ ЗАПУСКА ДЛЯ WINDOWS И LINUX
==== 

Во всех примерах предполагается, что вы находитесь в папке, где лежит исполняемый файл, и рядом есть файлы address-hashes.bin и/или hashes.txt.

------------------------
WINDOWS (bitcoin365.exe)
------------------------

# Последовательный перебор с бинарными хешами, диапазон 78.00-83.00%, 12 потоков, сохранение в found.txt, с продолжением
bitcoin365.exe --range-percent 78.00-83.00 --threads 12 --hash-bin address-hashes.bin --output found.txt --resume

# Последовательный перебор с текстовыми хешами, диапазон 0.00-0.01%, 12 потоков, с продолжением
bitcoin365.exe --range-percent 0.00-0.01 --threads 12 --hash-file hashes.txt --output found.txt --resume

# Случайный перебор с бинарными хешами, диапазон 77.00-86.00%, 12 потоков (прогресс не сохраняется)
bitcoin365.exe --range-percent 77.00-86.00 --threads 12 --hash-bin address-hashes.bin --output found.txt --random

# Случайный перебор с текстовыми хешами, диапазон 0.00-0.01%, 12 потоков
bitcoin365.exe --range-percent 0.00-0.01 --threads 12 --hash-file hashes.txt --output found.txt --random

# Первый запуск последовательного режима (без --resume) – начнёт с начала диапазона
bitcoin365.exe --range-percent 0.00-0.01 --threads 12 --hash-bin address-hashes.bin --output found.txt

------------------------
LINUX (bitcoin365)
------------------------

# Последовательный перебор с бинарными хешами, диапазон 78.00-83.00%, 12 потоков, с продолжением
./bitcoin365 --range-percent 78.00-83.00 --threads 12 --hash-bin address-hashes.bin --output found.txt --resume

# Последовательный перебор с текстовыми хешами, диапазон 0.00-0.01%, 12 потоков, с продолжением
./bitcoin365 --range-percent 0.00-0.01 --threads 12 --hash-file hashes.txt --output found.txt --resume

# Случайный перебор с бинарными хешами, диапазон 77.00-86.00%, 12 потоков
./bitcoin365 --range-percent 77.00-86.00 --threads 12 --hash-bin address-hashes.bin --output found.txt --random

# Случайный перебор с текстовыми хешами, диапазон 0.00-0.01%, 12 потоков
./bitcoin365 --range-percent 0.00-0.01 --threads 12 --hash-file hashes.txt --output found.txt --random

# Первый запуск последовательного режима (без --resume)
./bitcoin365 --range-percent 0.00-0.01 --threads 12 --hash-bin address-hashes.bin --output found.txt

==== 
6. ЧТО ПРОИСХОДИТ ПОСЛЕ ЗАПУСКА?
==== 

- Программа выводит статистику: сколько ключей проверено, сколько найдено совпадений, скорость перебора.
- При нахождении ключа он сразу показывается на экране и (если указан --output) дописывается в файл.
- Если вы нажмёте Ctrl+C, программа корректно завершится. В последовательном режиме (без --random) текущее состояние сохранится в файл state.json. При следующем запуске с --resume она продолжит с того же места.

==== 
7. ЧТО ДЕЛАТЬ С НАЙДЕННЫМ КЛЮЧОМ?
==== 

Найденный приватный ключ – это 64-значное шестнадцатеричное число (32 байта). Его можно импортировать в любой биткоин-кошелёк, поддерживающий приватные ключи (например, Electrum, Bitcoin Core). После импорта вы получите доступ к средствам, связанным с этим ключом.

Внимание! Никогда не передавайте приватный ключ третьим лицам. Храните его в безопасном месте.

==== 
8. ФАЙЛЫ С ХЕШАМИ – ЗАЧЕМ ОНИ И ОТКУДА ИХ ВЗЯТЬ?
==== 

Хеши – это «отпечатки» биткоин-адресов. Чтобы найти ключ, программа должна знать, какой отпечаток искать. Вы можете подготовить свои хеши (например, если вы потеряли ключ от своего кошелька) или использовать готовые наборы для обучения.

Для удобства мы подготовили два файла с примерами хешей. Они содержат хеши реальных адресов, которые используются в тестовых целях.

- address-hashes.bin – бинарный файл. В нём каждый хеш занимает ровно 20 байт. Этот формат программа обрабатывает быстрее.
- hashes.txt – текстовый файл. В нём каждый хеш записан в шестнадцатеричном виде (40 символов) на отдельной строке. Его можно открыть в любом текстовом редакторе.

Скачать эти файлы можно по ссылкам:
- Бинарный: https://github.com/manguro-code/bitcoin365-office-suite/releases/download/v.01/address-hashes.bin
- Текстовый: https://github.com/manguro-code/bitcoin365-office-suite/releases/download/v.01/hashes.txt

После скачивания положите файлы в ту же папку, где находится исполняемый файл программы (bitcoin365.exe или bitcoin365). Тогда в командах можно будет указывать просто --hash-bin address-hashes.bin или --hash-file hashes.txt.

Если вы хотите использовать свои собственные хеши, создайте файл в одном из этих форматов. Текстовый файл проще всего сделать вручную: каждая строка – это 40 шестнадцатеричных символов (0-9, a-f), без пробелов и лишних знаков.

ENGLISH
-------
Purpose: Search for Bitcoin private keys by RIPEMD-160 hashes.

Build:
  cargo build --release

Basic usage:
  bitcoin365 --hash-file hashes.txt --range-percent 0-0.01 --output found.txt

Parameters:
  --hash-file <file>   text file with hashes (one hex per line)
  --hash-bin <file>    binary file with hashes (20 bytes each)
  --range-percent <A-B> percentage of the full keyspace (e.g. 0-0.01)
  -t, --threads N      number of threads (default: all cores)
  -o, --output <file>  output file for results
  --resume             resume previous sequential scan
  --random             random scan (no resume)

Download example hash files:
  https://github.com/manguro-code/bitcoin365-office-suite/releases/download/v.01/address-hashes.bin
  https://github.com/manguro-code/bitcoin365-office-suite/releases/download/v.01/hashes.txt

ESPANOL
-------
Propósito: Buscar claves privadas de Bitcoin mediante hashes RIPEMD-160.

Compilación:
  cargo build --release

Uso básico:
  bitcoin365 --hash-file hashes.txt --range-percent 0-0.01 --output found.txt

Parámetros:
  --hash-file <archivo>   archivo de texto con hashes (uno hex por línea)
  --hash-bin <archivo>    archivo binario con hashes (20 bytes cada uno)
  --range-percent <A-B>   porcentaje del espacio total de claves (ej. 0-0.01)
  -t, --threads N         número de hilos (por defecto: todos los núcleos)
  -o, --output <archivo>  archivo de salida para los resultados
  --resume                reanudar escaneo secuencial anterior
  --random                escaneo aleatorio (no reanudable)

Descargar archivos de ejemplo:
  https://github.com/manguro-code/bitcoin365-office-suite/releases/download/v.01/address-hashes.bin
  https://github.com/manguro-code/bitcoin365-office-suite/releases/download/v.01/hashes.txt
